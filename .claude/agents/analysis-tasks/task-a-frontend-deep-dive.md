# TASK A: Frontend Architecture Deep Dive

## **When to Use**
Project has significant frontend component

## **Objective**
Analyze frontend-specific patterns and technologies

## **Prerequisites**
- Completed basic architecture analysis
- Access to frontend source code and assets

## **Input Requirements**
- Project root path: `[SPECIFY PATH]`
- Frontend framework: `[FROM BASIC ANALYSIS]`
- Frontend directories: `[IDENTIFY FROM PROJECT]`

## **Task Instructions**

```
Analyze frontend architecture patterns in [PROJECT_PATH]. Focus on:

1. **Component Architecture and State Management**:
   - Component organization and hierarchy patterns
   - State management approach (Redux, Context, Vuex, etc.)
   - Component communication patterns
   - Props and event handling conventions

2. **Styling Patterns and Design Systems**:
   - CSS architecture (BEM, styled-components, CSS modules)
   - Design system implementation
   - Theme and variable management
   - Responsive design patterns

3. **Build Processes and Asset Management**:
   - Build tool configuration (Webpack, Vite, etc.)
   - Asset optimization strategies
   - Code splitting and lazy loading patterns
   - Bundle analysis and optimization

4. **Client-side Routing and Navigation**:
   - Routing library and patterns
   - Route protection and guards
   - Navigation state management
   - Deep linking and URL structure
```

## **Expected Deliverable**

Create a file named `frontend_architecture_patterns.md` containing:
- Component architecture documentation
- State management patterns
- Styling and design system guide
- Build and asset optimization strategies
- Routing and navigation patterns

## **Success Criteria**

- [ ] Component patterns clearly documented
- [ ] State management approach understood
- [ ] Build processes mapped
- [ ] Styling conventions identified
