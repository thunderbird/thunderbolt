## Decisions taken

- **Routing**: plan referenced `src/routes/settings/_layout.tsx` which doesn't exist; project uses `src/settings/*.tsx` for pages and `src/layout/sidebar/settings-sidebar.tsx` for nav. Followed plan's owned-files (`src/routes/settings/agents/index.tsx`) verbatim AND added the nav entry to the actual sidebar + registered the route in `src/app.tsx`. Minimal touch on app.tsx (one import + one Route).
- **iOS detection**: no pre-existing `isTauriIOS()` helper. Composed inline from `isTauri()` + `getPlatform() === 'ios'` (both from `src/lib/platform.ts`) and exposed via `validateAgentUrl(url, isIos)` so tests pass platform overrides without module mocks.
- **Built-in toggle**: rendered disabled (with tooltip explaining "always available") rather than a no-op enabled toggle. Visually clearer.
- **Delete confirmation**: matched MCP servers UX — a Popover with Cancel/Remove buttons. Same predicate (`canDeleteAgent`) used by the row's visibility check and the unit test.
- **Test strategy**: avoided `mock.module` per `docs/development/testing.md`. The dialog accepts an `isIos` DI prop; the list/row tests render with plain mocks of `onToggle`/`onDelete`. No PowerSync DB needed since the list is pure presentation.
- **Transport inference**: `inferTransport` exported from the dialog so M2 / future validation can reuse the same source of truth.
